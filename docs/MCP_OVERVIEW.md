# MCP (Model Context Protocol) — Обзор системы инструментов

## Что такое MCP?

MCP (Model Context Protocol) — это стандартный протокол, который позволяет LLM-моделям безопасно и структурированно получать доступ к внешним инструментам: файловой системе, браузеру, API, терминалу и другим системам.

Протокол разделил инструменты на три категории:
| Тип | Описание |
|-----|----------|
| **tools** | Кастомные инструменты (функции) с входными/выходными параметрами |
| **resources** | Данные из внешних источников (файлы, API, веб-страницы) |
| **prompts** | Интерактивные шаблоны для взаимодействия с пользователем |

### Компоненты

1. **MCP Гейтвей** — центральная точка, через которую все запросы проходят
2. **Сервера инструментов** — реализуют конкретные инструменты (chrome, GitHub, SSH)
3. **LLM-модель** — принимает контекст с инструментами и генерирует ответы

## Конфигурация серверов

Файл `mcp.json` определяет все подключённые сервера:

### chrome-devtools
```json
{
  "command": "/home/slabak/Desktop/start-chrome-devtools.sh",
  "args": [],
  "directTools": true
}
```
- **Назначение**: Инструменты для работы с Chrome DevTools (снимки страниц, паузы в JS, heap snapshots)
- **directTools**: true — инструменты доступны напрямую без префикса `server.`

### github
```json
{
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-github"],
  "directTools": ["search_repositories", "get_file_contents"]
}
```
- **Назначение**: Поиск репозиториев и получение содержимого файлов на GitHub
- **directTools**: только два инструмента доступны напрямую — остальные с префиксом `server.github.`

### ssh-mcp
```json
{
  "command": "npx",
  "args": [
    "ssh-mcp",
    "-y",
    "--host=2.26.67.89",
    "--port=2091",
    "--user=vibecoder",
    "--password=123123",
    "--timeout=300000",
    "--maxChars=none"
  ]
}
```
- **Назначение**: SSH-клиент с поддержкой MCP — удалённый доступ к серверу
- **Параметры**: host, port, user, password, timeout, maxChars

## Настройки в mcp.json

| Параметр | Значение по умолчанию | Описание |
|----------|----------------------|----------|
| `toolPrefix` | `server.` | Префикс для инструментов с GitHub сервера |
| `idleTimeout` | `10` (сек) | Время неактивности до закрытия соединения |
| `requestTimeoutMs` | `30000` | Таймаут на запрос к серверу |
| `directTools` | `true` | Разрешить прямой доступ к инструментам без префикса |
| `disableProxyTool` | `false` | Блокировать прокси-инструменты (если true) |
| `autoAuth` | `true` | Автоматическая авторизация при подключении |

## Как использовать инструменты

### Прямые вызовы (без префикса)

```javascript
// Для chrome-devtools и ssh-mcp
chrome_devtools_take_snapshot()
ssh_mcp_exec(command)
```

### С префиксом (GitHub сервер)

```javascript
// Для github сервера — нужен префикс server.github.
server.github_search_repositories(query)
server.github_get_file_contents(owner, repo, path)
```

## Типы вызовов инструментов

| Тип | Описание | Пример |
|-----|----------|--------|
| **Simple** | Возвращает строку/объект | `get_weather()` → `"23°C"` |
| **Streaming** | Поточный ответ | `read_file()` — стримит по байтам |
| **Progressive** | Возврат промежуточных результатов | `git_commit()` — шаг за шагом |

## Ограничения и best practices

1. **Таймауты**: Устанавливайте разумные таймауты (30-60 сек для HTTP, 5-10 мин для SSH)
2. **Пароли**: Никогда не храните пароли в чистом виде — используйте `ssh-mcp` с `-p`
3. **Токенизация**: MCP автоматически токенизирует ответы — длинные файлы могут уходить в контекст
4. **Error handling**: ЛLM должна обрабатывать ошибки серверов самостоятельно

## Валидация конфигурации

Проверьте, что все серверы подключены корректно:

```bash
# Проверка npx (для github и ssh-mcp)
npx --version

# Проверка скрипта chrome-devtools
chmod +x /home/slabak/Desktop/start-chrome-devtools.sh
/home/slabak/Desktop/start-chrome-devtools.sh
```

## Обновление конфигурации

1. Измените `mcp.json` — параметры серверов, timeouts и т.п.
2. Перезапустите pi-процесс: `pi -r` или перезагрузите терминал
3. Сервера переподключатся автоматически при изменении параметров
