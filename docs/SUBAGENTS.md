# Подпеденты (Subagents) — Руководство по использованию

## Что такое подпедент?

Подпедент — это автономный агент, который работает внутри основного агента. Каждый подпедент может:
- Выполнять свои задачи независимо
- Получать контекст от родителя (`{previous}` — предыдущий ответ)
- Использовать те же инструменты и навыки
- Обмениваться сообщениями через `intercom`

## Архитектура

```
┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│   Основной    │◀───▶│ Подпедент 1   │◀───▶│ Подпедент 2   │
│      агент    │     │ (documenter)  │     │ (researcher)  │
└───────────────┘     └───────────────┘     └───────────────┘
```

## Управление подпедентами через subagent_supervisor

### Список доступных агентов

```bash
subagent_supervisor({ action: "list" })
# или
mcp({ action: "list", server: "github" })
```

### Отображение деталей агента

```bash
subagent_supervisor({ action: "get", agent: "documenter" })
```

### Отправка сообщения подпеденту

```bash
subagent_supervisor({
  action: "send",
  to: "documenter",
  message: "Проверь API документацию для функции get_weather()"
})
```

### Ответ подпеденту (из родителя)

```bash
subagent_supervisor({
  action: "reply",
  replyTo: "msg12345",
  message: "Вот результат проверки..."
})
```

### Просмотр ожидающих задач

```bash
subagent_supervisor({ action: "pending" })
# или
mcp({ action: "pending" })
```

### Статус выполнения задачи

```bash
subagent_supervisor({ action: "status", id: "run-abc123" })
```

## Примеры использования подпедентов

### 1. Исследователь + Документатор

**Задача**: Создать документацию для нового API

**План**:
1. `researcher` — ищет информацию об API
2. `documenter` — пишет документацию на основе данных

```json
{
  "chain": [
    { "agent": "researcher", "task": "{task}" },
    { "agent": "documenter", "task": "Напиши документацию по результатам исследования" }
  ]
}
```

### 2. Junior-кодер + Tester

**Задача**: Написать и протестировать функцию

**План**:
1. `junior-coder` — пишет код
2. `tester` — проверяет корректность

```json
{
  "chain": [
    { "agent": "junior-coder", "task": "{task}" },
    { "agent": "tester", "task": "Проверь написанный код" }
  ]
}
```

### 3. Параллельные подпеденты (fan-out)

**Задача**: Исследовать несколько источников одновременно

```json
{
  "tasks": [
    { "agent": "researcher", "task": "Исследуй GitHub", "count": 2 },
    { "agent": "documenter", "task": "Собери документацию" }
  ]
}
```

### 4. Асинхронные подпеденты (async)

**Задача**: Выполнить долгую задачу на фоне

```json
{
  "agent": "researcher",
  "task": "{task}",
  "async": true,
  "output": "/home/slabak/.pi/research_output.md"
}
```

## Типы вызовов подпедентов

### SINGLE (одиночный)

Выполняется один подпедент — идеален для простых задач.

```json
{ "agent": "documenter", "task": "Напиши документацию по функции" }
```

### CHAIN (цепочка)

Подпеденты выполняются последовательно — результат одного становится входным для следующего.

```json
{
  "chain": [
    { "agent": "researcher", "task": "{task}" },
    { "agent": "documenter", "task": "Напиши документацию" }
  ]
}
```

### PARALLEL (параллельно)

Несколько подпедентов работают одновременно.

```json
{
  "tasks": [
    { "agent": "researcher", "task": "{task}", "count": 3 },
    { "agent": "documenter", "task": "{previous}" }
  ]
}
```

## Настройки выполнения

### Таймауты

- `defaultTimeoutMs` — общий таймаут для всех подпедентов (из settings.json)
- `timeoutMs` — отдельный таймаут (в мс)
- `maxRuntimeMs` — альтернатива timeoutMs

### Контекст

| Value | Описание |
|-------|----------|
| `"fresh"` | Свой контекст — не использует данные родителя |
| `"fork"` | Ветвление от родительского контекста (по умолчанию) |

### Ограничения инструментов

```json
{
  "toolBudget": {
    "hard": 10,      // Жёсткий лимит попыток
    "soft": 5,        // Мягкое предостережение
    "block": ["*"]    // Блокировать все инструменты (или список)
  }
}
```

### Ограничения по ходу выполнения

```json
{
  "turnBudget": {
    "maxTurns": 10,   // Макс. ходов LLM
    "graceTurns": 2   // Бонусные ходы перед завершением
  }
}
```

## Приёмы работы с подпедентами

### 1. Разделение ответственности

Не давайте одному агенту делать всё. Используйте несколько специализированных агентов:
- Один для поиска информации
- Один для написания кода
- Один для тестирования
- Один для документирования

### 2. Чёткие инструкции в задаче

```json
{
  "task": "Напиши функцию, которая возвращает текущую дату и время в формате ISO 8601"
}
// vs
{
  "task": "{task}"
}
```

### 3. Использование {previous}

Всегда передавайте предыдущий ответ — это контекст для следующих шагов.

### 4. Асинхронность для долгих задач

Запускайте подпеденты в фоне с `async: true` и отслеживайте прогресс через supervisor.
